# Виктор Цой — Режиссёрский тритмент (HTML/CSS/JS)

Чистая статика. **Node / npm больше не нужны.** Весь проект — несколько файлов и папка с медиа. Шрифты и видео встроены локально, интернет для показа не требуется.

## Как посмотреть

Самый простой способ — **двойной клик по `index.html`** (откроется в браузере; в Chrome/Edge видео и шрифты подхватятся сразу).

Если в вашем браузере локальные файлы видео не проигрываются, поднимите мини-сервер в этой папке:
```
python -m http.server 8000
```
и откройте `http://localhost:8000`. Листание слайдов — стрелки ← → или точки внизу.

## Структура

```
index.html      — разметка, 8 экранов
styles.css      — все стили (палитра/эстетика архива, адаптивная типографика)
script.js       — навигация (свайп/колесо/клавиши) + ленивое воспроизведение видео
fonts.css       — локальные шрифты (Oswald, Cousine, Caveat; кириллица+латиница)
fonts/          — файлы шрифтов (.woff2)
media/          — все видео и постеры (кадры-заставки); media/_orig — бэкап исходных видео
```

8 экранов: 1) Главная, 2) Интро/обращение, 3) Основная линия работы (архив → коллажная анимация → реконструкция), 4) Визуальный язык архивов (Archive Language), 5) Визуальный язык реконструкции (Reconstruction Language), 6) Система переходов, 7) Инструменты производства, 8) Концовка.

### Навигация и адаптивность

- **Листание:** свайп вверх/вниз (или влево/вправо) на телефоне, колесо мыши, стрелки ← → ↑ ↓ (и PageUp/Down, Home/End), кнопки и точки внизу. Поддержан deep-link: адрес `…/#s6` открывает сразу нужный экран.
- **Типографика** масштабируется по ширине экрана (`clamp()` + `vw`) — корректно и на телефоне, и на большом мониторе; жёстких `px` в тексте больше нет.
- Длинные текстовые экраны при нехватке высоты аккуратно прокручиваются внутри слайда (контент не уходит под навигацию и не обрезается). Учитываются «вырезы» экрана (notch / safe-area) и `prefers-reduced-motion`.

## Как заменить / добавить медиа

1. Положите файл в `media/` (например `media/new.mp4`).
2. В `index.html` найдите нужную видео-ячейку — это тег вида:
   ```html
   <video poster="media/archive-cassette-poster.jpg" muted loop playsinline preload="none" data-src="media/archive-cassette.mp4"></video>
   ```
3. Поменяйте `data-src` на свой файл. Для красивой заставки сделайте кадр-постер:
   ```
   ffmpeg -i media/new.mp4 -vf "thumbnail" -frames:v 1 media/new-poster.jpg
   ```
   и пропишите его в `poster="media/new-poster.jpg"`.

Видео проигрываются без звука, по кругу, и только на активном экране (экономия ресурсов).

## Какой клип где стоит (чтобы было удобно менять местами)

- **Экран 1 (фон):** `cover.mp4` (есть запасной `cover-alt.mp4`)
- **Экран 2 (фон):** `intro.mp4`
- **Экран 3:** Архив `archive-cassette.mp4` · Коллаж `archive-collage.mp4` · Реконструкция `recon-cutout.mp4`
- **Экран 4 (коллаж справа):** `archive-cassette.mp4`, `archive-cutouts.mp4`, `archive-loop.mp4`
- **Экран 5 (4 карточки):** Cut-out `recon-paper.mp4` · Sketch `recon-main.mp4` · Punk `recon-punk.mp4` · Paper `recon-storyboard.mp4`
- **Экран 6 (фон):** `archive-collage.mp4` · **Экран 7 (фон):** `recon-cutout.mp4` · **Экран 8 (фон):** `cover.mp4`
- Не используются сейчас: `recon-cutout.mp4` (только эк.3), `recon-good.mp4`, `recon-tving.mp4`

> Видео — **сжатые для веба** версии (без звука, зацикленные). При доработке: у роликов с запечёнными чёрными рамками рамки **обрезаны** (`crop`), а тяжёлые файлы пересжаты (`recon-storyboard.mp4` 42 МБ → ~6.8 МБ, `recon-punk.mp4` 6.8 → 2.0 МБ). Все видео отображаются через `object-fit: cover` (без полей). Исходники до обработки лежат в `media/_orig/` — папку можно удалить, если бэкап не нужен.

> **Концовка (экран 8):** ссылка на доску Miro сейчас — заглушка `https://miro.com/`. Замените её на реальный адрес: в `index.html` найдите `<a class="btn" href="https://miro.com/"` и подставьте свою ссылку.

## Как поделиться с заказчиком

Это статический сайт — подойдёт любой хостинг статики, ссылка за пару минут:
- **Vercel / Netlify / Cloudflare Pages** — перетащить папку, получить публичный адрес.
- **GitHub Pages** — залить файлы как есть (здесь `base` уже не нужен — нет сборки и абсолютных путей).

Поскольку всё локально (шрифты, видео), сайт одинаково работает и онлайн, и офлайн.
